# MarineAI – Droid-Driven Development Plan  
_Leveraging AI assistants (‛Droids’) to accelerate, harden, and document every stage of the MarineAI vessel-manual platform._

---

## 1. Guiding Principles for Droid Usage
1. **Human-in-the-Loop** – Droids generate, refactor, and review code; humans approve, merge, and own decisions.  
2. **Automate the Boring, Elevate the Complex** – Routine scaffolding, tests, docs, and CI steps are delegated to droids so engineers focus on domain logic and UX.  
3. **Observable & Reproducible** – All droid actions (prompts, outputs, commits) are logged and traceable in the repo and CI systems.  
4. **Security & Compliance First** – Droids run in least-privilege sandboxes; sensitive credentials never leave controlled environments.

## 3. Droid-Assisted Development Workflow

1. **Design Kick-off**  
   a. Architect drafts high-level spec in Markdown.  
   b. Doc Droid converts spec → ADR template & PlantUML diagram.  
2. **Feature Ticket Creation**  
   – Product Owner creates GitHub issue → `/droid plan` comment auto-generates subtasks & acceptance criteria.  
3. **Branch & Scaffolding**  
   – `/droid scaffold nestjs module Vessel` ← Code Droid generates controller, service, DTOs, tests.  
4. **Daily Coding Loop**  
   - Developer writes prompt-driven tasks inside IDE.  
   - Code Droid suggests code; Test Droid mirrors with Jest specs.  
   - Local pre-commit hook runs Review Droid (eslint, prettier, CodeQL-AI).  
5. **Pull Request Phase**  
   - PR template includes `/droid review` toggle.  
   - Review Droid leaves inline comments, security alerts, test-coverage diff.  
   - Human reviewer approves/requests changes.  
6. **Merge & CI**  
   - DevOps Droid updates Helm chart versions.  
   - Test Droid executes full Jest & Playwright suites in GitHub Actions.  
7. **Release & ChatOps**  
   - `/deploy staging` in Slack triggers ArgoCD sync.  
   - ChatOps Droid posts health check summary & regression diff.

## 4. AI-Powered Code Generation Strategies

| Area | Prompt Pattern | Output |
|------|----------------|--------|
| NestJS CRUD modules | “Generate NestJS module for ‹Entity› with TypeORM repo, DTO validation, OpenAPI docs” | Boilerplate + unit tests |
| React Hooks & UI | “Build React hook for WebSocket chat with optimistic updates; 100% TypeScript” | `useChat.ts`, Storybook story |
| FastAPI endpoints | “Create FastAPI `/ingest` endpoint with Pydantic schema & background task using RQ” | route + worker task |
| Infrastructure | “Draft Helm chart for AI-service with env var secrets & HPA” | `charts/ai-service` |
| SQL Migrations | “SQL migration: add ‘quality_score’ decimal to pages table; reversible” | Alembic script |

Best Practices:  
- Always generate _and_ immediately run Test Droid on the output.  
- Constrain droid context with only relevant code to avoid hallucinations.  
- Commit prompts alongside generated code for auditability (`/prompts` folder).

---

## 5. Automated Testing & Validation

1. **Unit Tests** – Generated by Test Droid for every module/function; enforced ≥ 90 % threshold.  
2. **Contract Tests** – Postman or Pact-Broker with AI-generated specs for inter-service APIs.  
3. **AI Regression Harness** – Golden-set of user queries → expected answers; AI Service tested nightly to detect drift.  
4. **Visual Regression** – Playwright-AI takes baseline screenshots of PDF viewer & chat UI.  
5. **Load & Chaos** – k6 + Infra-GPT create scripts; SRE droid schedules chaos tests in staging.

## 7. Code Quality & Consistency Measures

- **Style Guides**: ESLint/Prettier + Ruff + Black; Review Droid blocks non-compliant PRs.  
- **Semantic Commits**: Commitizen with `/droid commit --type feat` wizard.  
- **Static Analysis**: CodeQL-AI, Bandit, TypeScript strict mode.  
- **Dependency Updates**: Renovate with AI-generated upgrade PR summaries.

## 8. Continuous Integration / Delivery Enhancements

1. **Pipeline Generation** – DevOps Droid authors reusable GitHub Actions YAML templates.  
2. **Smart Caching** – AI predicts build graph to cache Docker layers & `node_modules`.  
3. **Dynamic Test Selection** – Test Droid chooses impacted tests via changed files map.  
4. **Review Gates** – PR merges require ✅ from Review Droid + human.

## 9. Documentation Generation & Maintenance
